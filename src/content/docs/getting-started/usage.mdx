- Inicie um repositório git vazio

```bash
mkdir kuma-hello-world && cd kuma-hello-world && git init
```

- Adicione seu primeiro módulo com o Kuma CLI

```bash
kuma module add --repository kuma-framework/kuma-hello-world
```

- Siga os passos em seu terminal até que seja exibida a mensagem de "Hello World!"

Se você seguiu corretamente os passos anteriores, você deverá ter o seguinte diretório:
<FileTree>
- .kuma
    - kuma-hello-world
        - runs
            - hello-world-runs.yaml 
        - templates
           -  Main.go
        - base.yaml
    - kuma-modules.yaml
- go.mod
- main.go
- .gitmodules
</FileTree>

Todos os arquivos do kuma ficam dentro da pasta `.kuma`. Dentro dessa pasta, a separação é feita por módulos, (mas você também pode criar runs que não pertencem a nenhum módulo).
O módulo `kuma-hello-world` é o primeiro módulo que você adicionou ao seu projeto.
Dentro do módulo, você encontrará os seguintes diretórios:

- `runs`: esse diretório contém os arquivos de configuração de runs, que são arquivos YAML que definem o processo anterior a aplicação dos templates para gerar o código. [Saiba mais](/overview/runs/)
- `templates`: esse diretório contém os arquivos de template que serão usados para gerar o código. [Saiba mais](/overview/templates/)

O arquivo `base.yaml` é arquivo responsável por definir a estrutura de diretórios e arquivos que serão criados em seu projeto. [Saiba mais](/overview/builders/)

O arquivo `kuma-modules.yaml` é o arquivo de configuração do Kuma, ele contém informações sobre os módulos que você adicionou ao seu projeto e é modificado automaticamente sempre que um módulo for adicionado ou removido.

Cada módulo no Kuma, é também um submódulo do git. Por conta disso, um arquivo `.gitmodules` foi criado na raiz de seu projeto. [Saiba mais](/overview/modules/)

Os arquivos `go.mod` e `main.go` foram gerados através da execução da `run` **initial**
