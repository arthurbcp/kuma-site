import { FileTree } from '@astrojs/starlight/components';


- Inicie um repositório git vazio

```bash
mkdir hello-world && cd hello-world && git init
```

- Adicione seu primeiro módulo com o Mr. Smith CLI

```bash
mr module add --repository mr-smith-org/hello-world
```

- Siga os passos em seu terminal até que seja exibida a mensagem de "Hello World!"

![hello](https://github.com/user-attachments/assets/bf9c603f-1f58-4cde-ae9a-935bc6857f2f)



Se você seguiu corretamente os passos anteriores, você deverá ter o seguinte diretório:
<FileTree>
- .mr-smith
    - hello-world
        - runs
            - hello-world-runs.yaml 
        - templates
           -  Main.go
        - base.yaml
    - mr-modules.yaml
- go.mod
- main.go
</FileTree>

Todos os arquivos do Mr. Smith ficam dentro da pasta `.mr-smith`. Dentro dessa pasta, a separação é feita por módulos, (mas você também pode criar runs que não pertencem a nenhum módulo).
O módulo `hello-world` é o primeiro módulo que você adicionou ao seu projeto.
Dentro do módulo, você encontrará os seguintes diretórios:

- `runs`: esse diretório contém os arquivos de configuração de runs, que são arquivos YAML que definem o processo anterior a aplicação dos templates para gerar o código. [Saiba mais](/overview/runs/)
- `templates`: esse diretório contém os arquivos de template que serão usados para gerar o código. [Saiba mais](/overview/templates/)

O arquivo `base.yaml` é arquivo responsável por definir a estrutura de diretórios e arquivos que serão criados em seu projeto. [Saiba mais](/overview/builders/)

O arquivo `mr-modules.yaml` é o arquivo de configuração do Mr. Smith, ele contém informações sobre os módulos que você adicionou ao seu projeto e é modificado automaticamente sempre que um módulo for adicionado ou removido.

Os arquivos `go.mod` e `main.go` foram gerados através da execução da `run` **initial**
